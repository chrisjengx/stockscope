## 发现 1: `_direction_health` 仅用符号不用幅度

### 所在位置

[focus_list.py:364-379](backend/focus_list.py#L364-L379)

### 当前算法

```
判断 (d60, d20, d5, d3) 每个值的正负符号，分情况给 base 分:
  d60>0 且 d3>0 且 d5<=0 → base=75  (回调恢复, 给最高分)
  d60>0 且 d3>0 且 d5>0  → base=70  (正常上升)
  d60>0 且 d5>0          → base=60
  d60>0                  → base=45
  d3>0 且 d5>0           → base=60
  d3>0                   → base=30
  否则                     → base=10

加 sustained bonus:
  d20>0 且 d5>0 → +15
  d20>0         → +10
  否则           → 0

减 declining penalty:
  d20<0 且 d60<0        → -30
  d20<0 且 d3<=0        → -10
  否则                   → 0
```

### 问题

**只看符号，不看幅度。** d60=+0.1% 和 d60=+50% 在这个函数里没有区别。

推演三只股票：

| 股票 | d60 | d20 | d5 | d3 | 场景 | 得分 |
|------|-----|-----|-----|-----|------|------|
| A | +30% | +15% | +3% | +5% | 强趋势、无回调 | 70+15=**85** |
| B | +0.5% | +0.1% | -8% | +1% | 弱趋势、深回调中、刚反弹 | 75+10=**85** |
| C | +30% | +20% | -2% | +3% | 强趋势、浅回调、反弹中 | 75+10=**85** |

三只得分完全相同（85）。但：
- A 是稳健的持续上涨——它是好标的但非"早期"
- B 是弱趋势中一次可能只是死猫跳的反弹——d60=+0.5% 基本是横盘
- C 是强趋势中的健康回调——**最好的买入机会**

函数的设计意图是奖励"回调恢复"（B和C的 base=75 > A的70），这本身是对的——买回调比追涨好。但 B 不应该和 C 得分相同，因为 B 的"长期趋势"只涨了 0.5%（本质是横盘），回调深达 -8%，反弹仅 +1%。这更可能是下跌中继而非回调结束。

### 根因

`_direction_health` 的设计目标是对的——区分"回调买入机会"和"追涨"。但实现上缺少幅度信息，导致弱趋势中的假回调也被奖励。`_direction_structure`（同一模块中的另一个方向评估函数）也有同样的符号-only 限制，但两函数测量不同概念（模式 vs 健康度），所以各自独立存在是合理的。

### 影响路径

`_direction_health` → `_early_momentum_score`（权重 25%）→ hot_picks total_score（权重 50%）。对 hot_picks 总分的影响权重约为 12.5%。

### 修复方向

在 `_direction_health` 中引入幅度：将 base 分乘以一个基于 d60 幅度的缩放因子。例如：

```
# d60 幅度缩放: 强趋势(>20%)→1.0, 中等(5-20%)→0.9, 弱(<5%)→0.7
if d60 > 20: magnitude_scale = 1.0
elif d60 > 5: magnitude_scale = 0.9
else: magnitude_scale = 0.7

base = base * magnitude_scale
```

效果:
- 股票 B (d60=0.5%): base=75×0.7=52.5, 总分=52.5+10=**62.5**（从85降到63）
- 股票 C (d60=30%): base=75×1.0=75, 总分=75+10=**85**（不变）
- 股票 A (d60=30%): base=70×1.0=70, 总分=70+15=**85**（不变）

## 发现 2: long_term gate 的二元判断盲区

### 所在位置

[focus_list.py:164-165](backend/focus_list.py#L164-L165)

### 当前算法

```python
if (d20 or 0) < 0 and (d60 or 0) < 0:
    return False, "下跌趋势"
```

两个条件必须**同时为负**才拒绝。

### 问题

d20=+0.1%（勉强正）且 d60=-15%（明确下跌）→ **通过**。  
d20=-0.1% 且 d60=+30%（长期上升、短期微调）→ **被拒绝**。

第二种情况是误杀——长期上升趋势中的正常回调被拒绝。第一种情况是漏网——长期下跌趋势中的微弱反弹被放行。

对于 2-4 周持仓，d60 的方向比 d20 更有预测力：d60 大幅为负说明这只股票处于长期下跌通道中，d20 的微正大概率只是短期反弹。

### 修复方向

给 d60 更高权重：

```
weighted_trend = (d20 or 0) * 0.4 + (d60 or 0) * 0.6
if weighted_trend < -3:
    return False, f"下跌趋势(加权{weighted_trend:.1f}%<-3%)"
```

效果:
- d20=+0.1%, d60=-15% → 加权=-8.9% < -3% → **拒绝** ✓
- d20=-0.1%, d60=+30% → 加权=+17.9% > -3% → **通过** ✓  
- d20=-5%, d60=-5% → 加权=-5% < -3% → **拒绝** ✓（原逻辑也拒绝）
- d20=+5%, d60=+5% → 加权=+5% > -3% → **通过** ✓（原逻辑也通过）

## 影响汇总

| 发现 | 影响范围 | 严重度 |
|------|---------|--------|
| _direction_health 符号-only | hot_picks 总分约12.5%权重被弱趋势股虚高 | 中等 |
| Gate 二元判断盲区 | 长期下跌+短期微反弹的股票漏过 long_term 门控 | 中等 |

两个问题都是低频场景——需要特定的数值组合才会触发。不像第一轮发现的"确信度脱节"那样每次运行都影响所有股票。但它们会导致特定类型的错误：买入假反弹的股票（方向健康度假阳性）或漏过真回调的股票（gate 误杀）。

---

## 与第一轮的关系

第一轮修复解决了**系统性偏差**（每次运行都错）。本轮发现的是**条件性偏差**（特定数值组合下才错）。第一轮修复后，系统的大方向是正确的；本轮的两个修复让边缘情况也更精确。
