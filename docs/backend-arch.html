A0 — Universe Manager (股票分级)

决策树 (优先级从上到下)

1. EXCLUDED: ST/退市 | 60日均量<1000万 | 上市<60天
2. HOLDING: 已在 portfolio 持仓
3. AVOID: d5<0 & d20<0 & d30<0 (三周期同步下跌)
4. AVOID: MA60以下占比≥70% & MA空头 & 近20日低点 (持续阴跌)
5. classify_stocks (剩余):
   Path 1 (rule_direct): FAVORED=chg_20d>0 & chg_60d>0 | AVOID=流动性<3000万/振幅>80%
   Path 2 (LLM 30% 随机采样): FAVORED/NEUTRAL/AVOID
   Path 3 (rule_default): FAVORED=bullish & vs_ma60>0 | AVOID=bearish & vs_ma60<0 | NEUTRAL=其余
输入
• stocks: 全量代码 + ST标记 + 上市日期
• daily_quotes: close/volume/amount 120日
• portfolio: 当前持仓
• tier_assignments: 上一轮分级结果
输出
• tier_assignments: {ts_code, tier, reason, confidence, source, updated_at}
• 5个 Tier: EXCLUDED→HOLDING→FAVORED→NEUTRAL→AVOID
• 下游: A1 (只处理 H+F+N)

A7 — Portfolio Constructor (持仓构建, 核心)

三层架构

Layer 1: Conviction (纯数学, 零 LLM) — 策略分叉权重
  long_term: base(0.25) + mom_quality(0.25) + a2(0.15) + fragility(0.15) + penalty(0.20)
  hot_picks: base(0.25) + mom_quality(0.35) + a2(0.08) + fragility(0.12) + penalty(0.20)

Layer 2: LLM 选股 (分批, 15只/批, 7-15%纳入)
  每批得到: 宏观环境 + 板块资金偏好 + 市场新闻 + A5趋势分析 + 策略差异化角色

Layer 3: 硬约束 (安全网)
  权重≥min, ≤max | 总仓位≤1-min_cash | 安全上限25只

Bounded Authority (策略分叉)

Tierlong_termhot_picksLLM 权限
STRONGconv ≥ 0.50conv ≥ 0.45必须纳入, 8-25%
MODERATEconv ≥ 0.30conv ≥ 0.25可选, 8-15%
WEAKconv < 0.30conv < 0.25自动剔除

LLM 角色 (策略分叉)

long_term — 价值侦察兵
持仓 2-4 周 · 选股优先级: 趋势质量 > 基本面支撑 > 估值合理性
HIGH 红旗 → 建议 REJECT
hot_picks — 热点猎手
持仓 3-5 天 · 选股优先级: 动量强度 > 量价配合 > 基本面(仅查硬伤)
硬规则: d3<0 且 d5<-5 且 acc<0 → 必须 REJECT

Conviction 动量质量级联

d3>0, d5>0, acc>5  → 1.00  (确认上升+加速 — 最强信号)
d5>0, acc>5        → 0.90  (上升+加速)
d3>0, d5>0         → 0.80  (确认上升)
d3>0, acc>5        → 0.70  (早期反转信号 — 最强早期指标)
d3>0, acc>0        → 0.60  (趋势改善)
...
acc<-20            → 0.10  (强减速 — 回避)

A6 — Risk Officer (对抗审查)

双重审查 + 裁决

程序化检查 (6维度): 流动性 | 行业集中 | 持仓上限 | 估值极端 | 动量衰竭 | 信号冲突
LLM 深度审查: 读取 A1/A2/A3/A4/A5/A7 全部报告, 输出 risk_score 1-5

risk_score 校准锚:
  1=无可见风险 · 2=轻微疑虑 · 3=需关注 · 4=显著风险建议VETO · 5=致命风险强制VETO

裁决: risk≥4→VETO | 排名前60%→APPROVED | 排名后40%→VETO
      REJECT+低风险 → OVERRIDE (建议人工复核)
输入
• portfolio_decisions (A7产出)
• A1 indicators: MACD/RSI/MA/OBV
• A2 fundamental_reports: earnings/financial/red_flags
• A3 news_feed: per-stock + market background
• A4 macro_reports: regime_summary + sector_view + risk_alerts
• A5 fusion_reports + composite_scores
输出
• portfolio_decisions: 更新 status=APPROVED/VETOED/OVERRIDE_RECOMMENDED
• review_json: {a6: {risk_score, recommendation, reasoning, conditions, final_verdict}}
• risk_flags 表: 逐条风险标记

通用权重 (策略无关)

核心设计原则

  1. 定性替代定量: risk_budget 数字消除, 改为 regime_summary + sector_view + risk_alerts 语言描述。LLM 自己判断, 不用数字约束。
  2. 数学做安全网, LLM 做增值: Conviction 纯数学防止系统性错误, LLM 在安全边界内做定性判断。Bounded Authority 保证 LLM 不能越过数据边界。
  3. 板块是偏好不是约束: A4 sector_view→A7/A6, 同一段数据 A7 找机会、A6 找风险。LLM 自己判断, 没有硬性限制。
  4. 数据不重复加工: A1→A5 读 JSON 不重算。A2→A5→A7 单向流动。A4 一处产出、三处消费。
  5. 对抗决策: A7 乐观推荐, A6 悲观审查。A7 问"为什么涨", A6 问"A7 漏了什么"。
